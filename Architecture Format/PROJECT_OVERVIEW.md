## 3. General Development Principles

- **DRY (Don't Repeat Yourself)**: Tránh lặp lại mã, tận dụng reusability.
- **KISS (Keep It Simple, Stupid)**: Ưu tiên sự đơn giản và dễ hiểu.
- **SOLID**: Áp dụng 5 nguyên lý thiết kế hướng đối tượng/chức năng.
- **Clean Code**: Đặt tên có ý nghĩa, hàm đơn chức năng, tách biệt logic và UI.
- **Security First**: Luôn kiểm tra validation cho cả frontend và backend.

## 4. Communication Protocol

- **Client-Server**: RESTful API / GraphQL
- **Standard response**: `{ success: boolean, data: any, message: string }`
